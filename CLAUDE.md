## Project Overview

Wordwich is a browser-based word game. Players are given two "sandwich" letters and must submit the longest valid English word that starts with the first letter and ends with the second. Longer words score more points; beating the Master Word earns bonus points. Currently single-device only ("Host Game" multiplayer is UI-only and not wired to a backend).

## Tech Stack

- **Framework**: Vite + React 19 (web SPA, JSX)
- **State**: Zustand
- **Styling**: Tailwind CSS v4
- **Icons**: lucide-react
- **Dictionary validation**: bundled `src/data/words.json` (~180k words), checked in-memory via `Set`
- **Master word definitions**: fetched on demand from `api.dictionaryapi.dev`, cached in `localStorage`
- **Master word pool**: filtered to words that have a dictionary definition (so the Insight power-up always has something to show); list bundled at `src/data/qualified-master-words.json`, regenerated by `scripts/qualify-master-words.sh`

## Architecture

- `src/main.jsx` — Vite entry
- `src/App.jsx` — top-level screen router (no react-router; switches via Zustand state)
- `src/components/screens/` — screen components (Home, Difficulty, PlayerSetup, Game, Recap, etc.)
- `src/components/` — reusable UI (Timer, WordInput, Leaderboard, PowerUpBar, PlayerList)
- `src/hooks/` — custom hooks (useTheme, useTimer)
- `src/store/useGameStore.js` — Zustand store; all game state lives here
- `src/game/engine.js` — word validation, master/bridge word selection, dictionary fetch
- `src/game/scoring.js` — score calculation
- `src/data/words.json` — dictionary (large, ~2 MB raw)
- `src/data/qualified-master-words.json` — words eligible to be Master Words (have a dictionary definition)
- `scripts/qualify-master-words.sh` — regenerates the qualified list from the dictionary API

Word validation is encapsulated in `validateWord(word, startLetter, endLetter)` so the backing implementation can be swapped.

## Key Domain Concepts

- **Master Word**: The longest valid word for the given sandwich letters at the chosen difficulty; revealed in the recap screen
- **Round**: A single turn — letters shown, player submits a word, score is calculated
- **Score**: Based on word length; valid words only (must exist in `wordsSet`)

## Development Commands

```
# Install dependencies
npm install

# Dev server (Vite)
npm run dev

# Production build (outputs to dist/)
npm run build

# Preview the production build
npm run preview

# Lint
npm run lint

# Regenerate src/data/qualified-master-words.json (~22 min, hits api.dictionaryapi.dev)
npm run qualify:masters
```

## Deployment

Deployed to Vercel from `main` (auto-deploy on push). `vercel.json` at the repo root configures SPA rewrites and cache headers. `public/robots.txt` and the `noindex` meta tag in `index.html` keep the testing build out of search engines.
